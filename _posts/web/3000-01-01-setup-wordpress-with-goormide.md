## 구름 IDE 사용

지금은 깃허브 페이지에서 블로그를 운영하고 있지만 한때는 워드프레스를 직접 설치해보기도 했었다. 나만의 커스텀 테마 제작 욕구에 어설픈 지식과 실력으로 제작을 해본 적도 있다.

코딩도 편하게 하면서 워드프레스를 직접 설치해서 돌려보는데는 클라우드 IDE 만한게 없었다. 처음에는 Cloud 9 IDE 를 사용했지만 19년 여름에 완전히 [아마존 AWS](https://aws.amazon.com/ko/cloud9/)에 흡수가 되었는지 편하게 사용하기 어려워져서, 대안으로 찾은 것이 [구름 IDE](https://ide.goorm.io/)였다.

다만 아쉬운 점은 구름 IDE 는 워드프레스 개발을 위한 템플릿을 제공하지 않는다. 어쩔 수 없이 이런저런 설정을 직접 해가면서 설치할 수밖에 없다. 구글링도 하면서 오랜 시간 삽질 끝에 워드프레스 설치를 할 수 있었고 나중에 다시 필요할 수도 있기에 포스팅을 해 둔다.

사전에 독자들께 알려드리고 싶은 것은, **본인은 클라우드나 서버 혹은 웹개발 전문가도 아니고 하물며 관렵 업계 종사자도 아니다. 따라서 아래 내용들에 대한 정확성이나 안정성을 보장하지는 못한다.** 바꿔 말하자면 뭔가 이상해서 이런저런 것들을 본인에게 질문해도 제대로 대답조차 못할테니 미리 양해를 구하는 바이다.

## 컨테이너 생성

위 구름 IDE 링크를 타고 들어가서 회원가입을 한다. 이 부분은 굳이 설명하지 않는다.

회원가입 후 들어가면 우선 `새 컨테이너 생성`을 해야한다.

그러면 어떤 프로젝트를 위한 컨테이너를 생성할 것인지를 묻는데 다른 사항들은 건들지 않고 아래 사항만 설정한다.

- 이름: 원하는 이름, 본인은 워드프레스의 약자인 'wp' 라고 기입
- 소프트웨어 스택: 그림들 중 'php' 선택, template 은 'PHP 프로젝트', OS 는 'Ubuntu 18.04 LTS' 선택
- 추가 모듈/패키지: 다른 옵션들은 모두 비워두고 'MySQL 설치' 만 체크

여기까지 진행하면 **화면에 설치될 php 버전이 보이는데 따로 적어두자.**

이제 오른쪽 위에 보이는 `생성(CTRL+M)` 버튼을 클릭한다. 잠시 기다리면 설치 완료가 되고, `컨테이너 실행`을 클릭하면 아래와 같은 화면이 나타난다.

## 추가 설정

지금까지는 워드프레스 설치를 위한 php 서버가 준비되었을 뿐이다. IDE 내부를 보면 아래처럼 터미널 화면이 보이는데 여기에 이것저것 명령어를 순차적으로 입력해야 한다.

![터미널화면](https://zininote.github.io/img/web/web-0000-01-01-01.png)
{:.center}

터미널에 포커싱을 두고 아래 명령어들을 순서대로 입력한다. 명령어들을 입력할 때마다 뭔가 작업을 한다. 각 명령어들이 무엇을 의미하는지는 이 포스팅이 전달하고자 하는 바를 넘어서니 별도로 찾아보기를 바란다. (어차피 물어봐도 아는 게 없어 대답도 못할 것이다.)

- apt-get update
- service mysql start
- mysql -u root -p

여기까지 진행하면 패스워드를 입력하라고 하는데 아무 입력없이 그냥 엔터키만 눌러준다. 그러면 프롬프트가 `mysql>`로 바뀐다. 다시 아래 명령어들을 입력해주자. 세미콜론까지 정확하게 입력해야 한다.

- create database wordpress;
- exit

다시 원래의 프롬프트로 돌아오는데 이 시점에서 따로 적어두었던 php 버전을 확인한다. 본인이 설치할 땐 php 7.3.8 이었고 x.x 까지만 아래처럼 입력하면 된다.

- apt-get install php7.3-mysql

만일 위에서 버전을 따로 적어두지 못했다면 `php -v` 라는 명령어로 버전을 확인할 수 있다.

중간에 계속 진행하겠느냐는 물음이 나오는데 그냥 엔터를 누르면 된다. 이제 워드프레스를 설치할 차례다. 아래 명령어를 계속 수행한다.

- wget https://ko.wordpress.org/latest-ko_KR.tar.gz
- tar -zxvf latest-ko_KR.tar.gz

## 워드프레스 실행

이제 IDE 화면 위쪽에 보이는 `프로젝트` 메뉴의 `실행` 을 클릭하든가 아니면 `SHIFT+F5` 키를 눌러주자. 아래와 같은 화면이 나온다.

![실행화면](https://zininote.github.io/img/web/web-0000-01-01-02.png)
{:.center}

위 화면 오른쪽 아래에 뭔가 인터넷 주소같은 것이 보인다. **(이 주소) + /wordpress** 가 바로 클라우드에 설치된 워드프레스 주소가 된다.

참고로 주소 오른쪽에 있는 파란 버튼을 클릭하면 `(이 주소)` 까지만 복사가 되니 웹브라우저를 띄우고 거기에 `/wordpress` 만 더 추가하면 된다.

해당 주소로 들어가면 워드프레스가 작동한다. 이런저런 초기화를 하다가 아래와 같은 화면을 만나면 아래에 표시된 대로 입력을 해주자.

![워드프레스초기화](https://zininote.github.io/img/web/web-0000-01-01-03.png)
{:.center}

이후의 워드프레스 설정은 알아서 진행하면 된다.

## 추가 조치 사항

이 항목은 워드프레스를 사용하다가 문제가 생겼을 때 필요한 조치들을 정리했다.

플러그인 등을 설치할 때 아래 화면이 나오는 경우가 있다.

![에러메시지](https://zininote.github.io/img/web/web-0000-01-01-04.png)
{:.center}

일단 `취소`를 눌러 닫고, IDE 화면으로 돌아가서, 워드프레스가 설치된 폴더에서 `wp-config.php` 파일을 열고, 제일 아래에 다음과 같은 코드를 입력하고 저장한다.

```php
define('FS_METHOD', 'direct');
```

그리고 웹브라우저로 돌아가서 워드프레스를 다시 구동해본다.

또한 워드프레스 실행 중 데이터베이스에 접속할 수 없다는 에러를 보내올 때도 있는데, IDE 화면으로 돌아가서, 터미널에서 아래 명령어를 실행해 준다.

- service mysql start
