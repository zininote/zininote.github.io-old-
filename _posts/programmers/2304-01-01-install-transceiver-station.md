## 문제

[프로그래머스, 기지국 설치](https://programmers.co.kr/learn/courses/30/lessons/12979?language=python3)

문제 자체의 이해는 어렵지 않다. 기지국의 전파가 닿지 않는 아파트를 찾아서, 추가 기지국을 최소 몇개는 설치해야하는지를 탐색하면 된다.

처음에는 길게 늘어선 아파트를 인덱스로 표시한 리스트를 상정하고, 이 리스트에 전파가 닿는 아파트는 1, 그렇지 않는 아파트는 0 으로 표시한다음, 문자열로 합쳐 정규식으로 0 으로 이뤄진 문자만 분리해낸 뒤, 각각의 분리된 0 이 몇개인지에 따라 추가기지국 개수를 구하여 리턴했다.

이렇게 하니 정확성 테스트는 통과할 수 있었으나, 효율성 테스트는 통과할 수 없었다.

다른 방식으로 문제를 풀었다. `idx` 라는 일종의 포인터를 도입했다. 처음에는 1 번 아파트를 가리킨다. 그리고, 기지국을 반복하여 전파가 닿는 범위와의 계산을 통해, 전파가 안닿는 아파트들을 `a` 리스트에 저장한다. 그리고 이 리스트를 반복하여 추가기지국 개수를 구하여 리턴했다.

## 풀이

```py
def solution(n, stations, w):
    # 1
    a, idx = [], 1
    while idx < n+1:
        if stations:
            if idx < (s := stations.pop(0))-w:
                a.append((idx, s-w))
            idx = s+w+1
        else:
            a.append((idx, n+1))
            break
            
    # 2
    return sum(-(-(y-x)//(2*w+1)) for x, y in a)
```

`# 1` 에선, 전파가 안닿는 연속된 아파트가 예를들어 x 번부터 y 번까지라면 `(x, y)` 형태의 튜플로 만들어 `a` 리스트에 저장한다. python 의 일반적인 범위 규칙에 맡게 x 이상 y 미만 형태이다. 그리고 가장 앞에 있는 아파트를 가리키는 `idx` 변수를 생성하고 1 로 초기화한다.

반복문을 돌면서, 만일 `stations` 리스트에 남아있는 기지국이 있다면, `idx` < 이 기지국의 왼쪽 전파한계 라면 `idx` 부터 왼쪽 전파한계까지는 전파가 안닿는다는 의미이므로 이를 튜플로 만들어 `a` 에 삽입한다. 그리고 `idx` 를 이젠 오른쪽 전파한계까지 끌어올린다.

`stations` 에 더이상 남은 기지국이 없다면 `idx` 부터 `n` 까지는 모두 전파가 닿지 않는 아파트가 되므로 이를 `a` 에 담고 반복문을 탈출한다.

`# 2` 에서는 `a` 리스트를 반복하면서, 필요한 기지국 개수를 구하여 최종리턴한다. `sum` 함수 내부에 쓰인 `-(-(a//b))` 형태의 수식은 `math.ceil(a/b)` 와 동일한 결과를 보이는 수식이니 알아두면 좋다.

## 참고

효율성 테스트를 통과하지 못했던 코드도 아래에 나타냈으니 참고해보기 바란다.

```py
import re

def solution(n, stations, w):
    # 1
    a = ['0']*n
    for s in stations:
        for i in range(s-1-w, s+w):
            if -1 < i < n: a[i] = '1'
                
    # 2
    a = re.split(r'1+', ''.join(a))
    return sum(-(-len(x)//(2*w+1)) for x in a)
```
