## 문제

[프로그래머스, 구명보트](https://programmers.co.kr/learn/courses/30/lessons/42885?language=python3)

모든 사람들을 정원 2 명인 보트로 구출하는데 필요한 최소한의 보트수를 구하는 문제다. 일단 사람들을 무게 순으로 정렬한 뒤, 앞에서부터 한명씩 보트를 내어주되, 만일 여유가 있다면 뒤에서부터 보트를 얻어탈 수 있는 사람을 추가로 탐색하는 방식으로 해결해봤다.

참고로 효율성 테스트가 매우 빡빡한 문제다. 처음에는 `pop` 과 같은 함수를 사용했었는데, 리스트 삭제가 사용되면 효율성 테스트 통과를 못하는 것 같다.

## 풀이

```python
def solution(people, limit):
    # 1
    n = len(people)
    p = sorted(people)
    s = [0]*n
    
    # 2
    i, j = 0, n
    for i in range(n):
        if s[i] == 0:
            s[i] = 1
            for j in reversed(range(i, j)):
                if s[j] == 0 and p[i]+p[j] <= limit:
                    s[j] = 2
                    break
                else:
                    s[j] = 1
                    
    # 3
    return s.count(1)
```

`# 1` 에서 생성한 `s` 리스트는 일종의 helper 리스트다. `people` 을 오름차순으로 정렬한 `p` 의 동일한 인덱스의 사람이 배를 타기 전이면 0, 배를 탔다면 1, 만일 동승으로 얻어탔다면 2 를 배치한다. 나중에는 `s` 리스트에서 1 값을 가진 요소 개수를 리턴하면 된다.

`# 2` 에서는 우선 `i`, `j` 변수를 생성한다. 둘 모두 특정 인덱스를 가리키는 변수로, `i` 는 `p` 의 앞에서부터, `j` 는 뒤에서부터 탐색한다. 기본적으로 `p` 의 앞에서부터 보트를 내어준다. 즉 `s[i]` 에 1 을 할당한다. 보트를 받았으면 동승이 가능한지 여부를 탐색한다. `reversed` 함수를 사용하여 뒤부터 탐색토록 했다. 무게 합계가 `limit` 이내라면 `s[j]` 에는 2 를 할당한다.

 만일 `limit` 초과라면 즉시 `s[j]` 에는 1 을 할당한다. 그 이유는 보트는 어차피 가벼운 사람부터 할당을 받고, 동승자를 구하는 형태인데, 가벼운 사람과도 동승을 못했다면 다음 턴에서는 보다 무거운 사람과도 동승이 불가능하므로 어차피 따로 보트를 내줘야하기 때문이다.
