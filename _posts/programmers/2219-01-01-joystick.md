## 문제

[프로그래머스, 조이스틱](https://programmers.co.kr/learn/courses/30/lessons/42860?language=python3)

조이스틱의 최소한의 움직임을 좌우, 상하로 나눠서 파악하는 것이 핵심인 문제다.

개인적으로 좌우의 경우를 파악하는 것이 어려웠다. 조이스틱의 움직임을 좌, 우로 나눠서 재귀호출을 하는 방식으로 구해보았다. 상대적으로 상, 하 움직이는 횟수는 구현하기가 쉬웠다.

## 풀이

```python
def solution(name):
    # 1
    tmp = []
    def fn(m=[0], t=[i for i, x in enumerate(name) if x != 'A']):
        # 1-1
        if all(x in m for x in t):
            tmp.append(m)
            return
        if len(m) == len(name):
            return
        # 1-2
        left = m[-1]-1 if m[-1] != 0 else len(name)-1
        fn(m+[left], t)
        right = m[-1]+1 if m[-1] != len(name)-1 else 0
        fn(m+[right], t)
    
    # 2
    fn()
    a = min(len(x) for x in tmp)-1
    
    # 3
    b = sum(min(ord(x)-ord('A'), ord('Z')+1-ord(x)) for x in name if x != 'A')
    
    # 4
    return a+b
```

`# 1` 코드는 재귀호출로 조이스틱의 좌우 움직임을 구하기 위한 재귀함수다. 먼저 `tmp` 는 조건에 해당하는 조이스틱의 움직임을 담기 위한 리스트다. `fn` 함수의 인수를 보면, 첫번째 인수 `m` 은 조이스틱의 움직임이다. `name` 의 각 철자의 인덱스를 담을 것이며, 초기값으로 가장 왼쪽인 0 인덱스를 담고있다. `t` 는 `name` 에서 A 가 아닌 철자의 인덱스만을 담는다. 이 인덱스들은 조이스틱이 반드시 거쳐야하는 곳을 가리킨다.

`# 1-1` 코드는 재귀호출의 종료조건이다. 필수적으로 조이스틱이 거쳐야하는 `t` 의 각 요소가, 재귀호출로 조이스틱이 실제로 거쳐간 인덱스 `m` 에 모두 있는지를 먼저 검사한다. 그렇다면 이 조이스틱 움직임은 문제를 통과할 수 있는 움직임이므로 `tmp` 리스트에 추가한다. 그리고 문제의 조건은 좌우 움직임의 최소이며 `name` 길이 이상의 움직임은 최소 조건에서 무조건 벗어나므로 `m` 의 길이가 `name` 의 길이와 같아질 때 더 이상 재귀호출을 하지 않도록 한다.

`# 1-2` 코드는 직접 재귀호출을 하는 부분이다. 종료조건이 성립되지 않았다면 아직 조이스틱 움직임이 충분치 않다는 의미이므로 현재 시점 `m[-1]` 에서 좌로 움직이는 경우와 우로 움직이는 경우를 각각 호출한다.

`# 2` 코드에서는 `fn` 재귀함수를 호출한다. 그리고 `tmp` 안에 담겨있을 `name` 을 쓸 수 있는 조건을 만족하는 조이스틱 움직임 중에서 가장 길이가 짧은 것을 가져오고, 거기서 1 을 뺀다. 1 을 빼는 이유는 재귀함수 인수의 `m` 리스트 초기값인 0 은 조이스틱 움직이는 횟수에 포함되지 않기 때문이다.

`# 3` 코드는 상하 움직이는 횟수를 계산한다. A 인 경우는 상하 움직임이 필요없으므로 A 가 아닌 경우에 한하여 `sum` 함수 안에 있는 `min` 함수로 움직이는 횟수를 계산, 합계를 구한다.

`# 4` 에서 좌우 움직인 횟수인 `a` 와 상하 움직인 횟수인 `b` 를 더하여 리턴한다.
