## 문제

[프로그래머스, 캐시](https://programmers.co.kr/learn/courses/30/lessons/17680?language=python3)

문제에서 언급한 "LRU 캐시" 방식에 대한 이해가 있으면 그리 어렵지는 않은 문제다.

문제를 처음 접했을 때는 LRU 방식이 뭔지 몰라서 구글링을 했다. 아래와 같은 룰이 적용되는 것으로 이해하고 문제를 풀었다.

- 캐시는 Queue 와 같은 FIFO 자료형, 캐시 개수에 한계가 있을 수 있음
- 검색값이 캐시 안에 있으면 cache hit, 검색값을 queue 의 제일 뒷부분으로 옮김
- 검색값이 캐시 안에 없으면 cache miss, 검색값을 queue 의 제일 뒤에 추가

## 풀이

```py
from collections import deque

def solution(cacheSize, cities):
    # 1. 캐시 저장하는 cache deque 자료형 생성
    cache = deque(maxlen=cacheSize)
    
    # 2. 실행시간 a 생성, cities 반복, lru 캐시처리
    a = 0
    for city in cities:
        city = city.lower()
        if city in cache:
            a += 1
            cache.remove(city)
        else:
            a += 5
        cache.append(city)
        
    # 3. a 리턴
    return a
```

`# 1` 코드에서 캐시를 구현하기 위해 `collections` 모듈의 `deque` 자료형을 사용하였다. `deque` 자료형은 처음 생성할 때 `maxlen` 인수를 줄 수 있는데 저장공간의 최대크기이다. 편하게 LRU 작동방식을 구현할 수 있다. 만일 대신 일반 리스트를 사용한다면, 나중에 캐시를 추가할 때 최대크기에 맞춰 오래된 캐시를 삭제하는 구문을 따로 구현해줘야 한다.
