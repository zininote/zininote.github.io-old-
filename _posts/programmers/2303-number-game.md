## 문제

[프로그래머스, 숫자 게임](https://programmers.co.kr/learn/courses/30/lessons/12987?language=python3)

문제 자체의 이해는 어렵지 않다. 어떻게 해결해야하는지도 조금만 생각하면 쉽게 알 수 있는데, `A` 리스트 안의 수를 하나씩 가져와서 그 수를 간신히 이길 수 있는 `B` 안의 수와 매칭시키는 방법을 생각하면 된다.

예를들어, `A` 는 내림차순으로 정렬하고, `B` 는 오름차순으로 정렬한다음에, `A` 를 앞에서부터 순회한다. 그 숫자를 `B` 의 요소들과 하나하나 비교해 간다. 보다 큰 수가 나오는 순간 승리포인트를 올리고 그 숫자를 `B` 리스트에서 제거하는 방식으로 문제를 풀 수도 있다.

하지만 위 방식은 본 문제의 정확성 테스트는 통과할 수 있지만 효율성 테스트는 통과할 수 없다. `B` 리스트 요소와 비교할 때 순서대로 하나씩 비교하기 때문인데, [이진탐색](https://namu.wiki/w/%EC%9D%B4%EC%A7%84%20%ED%83%90%EC%83%89) 과 같은 알고리즘으로 비교를 해야 효율성 검사 통과가 가능하다. 이진탐색에 대해서는 [별도 포스팅]() 이나 포털검색으로 찾아보기 바란다.

python 에는 `bisect` 모듈안에 `bisect` 함수가 있는데, 어떤 숫자 x 가 오름차순으로 정렬된 리스트 안 어디에 들어가야 오름차순을 해치지 않는지를 인덱스로 알려주는 함수다. 이진탐색을 사용하여 인덱스를 찾아낸다. 이를 사용하여 문제를 해결했다. 궁금하신 분들은 [python 공식문서](https://docs.python.org/3.8/library/bisect.html)를 참고해보자.

## 풀이

```py
from bisect import *

def solution(A, B):
    # 1
    A.sort(reverse=True)
    B.sort()
    
    # 2
    point = 0
    for a in A:
        if (i := bisect(B, a)) < len(B):
            point += 1
            del B[i]
    
    # 3
    return point
```

`bisect` 모듈을 먼저 import 하였다.

`# 1` 에서는 위에서 언급한 대로 `A` 는 내림차순, `B` 는 오름차순으로 정렬한다.

`# 2` 를 보면, `A` 를 반복하면서, `bisect` 함수로 `B` 리스트 안에서 숫자 `a` 가 어디에 들어가야 오름차순을 해치지 않는지 인덱스를 `i` 로 받는다. 그 인덱스가 가리키는 숫자는 `a` 를 간신히 이길 수 있는 숫자가 된다.

만일 `B` 리스트 끝까지 가도 이길 수 있는 숫자가 없다면 `i` 는 `B` 리스트의 길이를 반환받는다. 이 경우라면 이길 수 있는 숫자가 없다는 뜻이다.

## 참고

위 코드에서 사용한 `bisect` 모듈의 코드가 궁금할 수도 있다. 그런 분들은 위에서 소개한 별도 포스팅이나 혹은 python 공식문서에서 공개한 [이 사이트](https://github.com/python/cpython/blob/3.8/Lib/bisect.py)를 들러보기 바란다.
