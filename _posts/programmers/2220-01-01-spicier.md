## 문제

[프로그래머스, 더 맵게](https://programmers.co.kr/learn/courses/30/lessons/42626?language=python3)

주어지는 리스트에서 최소값을 어떻게 효율적으로 빨리 찾아내는가가 문제 통과의 핵심이다.

최소값 찾기를 위해 최소 힙 자료구조를 사용해야 한다. 다른 방법은 대부분 효율성 테스트를 통과하지 못한다. 힙 자료구조에 대해서는 [별도 포스팅](업데이트예정)에 설명하였으니 관심있으신 독자들은 링크를 참고해보기 바란다.

python 은 힙 자료구조를 편리하게 사용할 수 있도록 `heapq` 모듈을 제공한다. 이를 사용하여 문제를 통과했다.

## 풀이

```python
from heapq import *

def solution(scoville, K):
    # 1
    s = scoville
    heapify(s)
    
    # 2
    a = 0
    while 1:
        # 2-1
        if s[0] >= K:
            return a
        if len(s) == 1:
            return -1
        
        # 2-2
        heappush(s, heappop(s) + heappop(s)*2)
        a += 1
```

`# 1` 에서는 `s` 리스트를 `heapify` 함수에 적용한다. 이렇게 하면 `s` 리스트는 heap 자료구조로 변형이 된다.

`# 2` 에서는 반복문을 통해 음식을 계속 섞어나가는 코드를 구현하였다. `a` 는 섞은 횟수를 나타내는 변수다.

`# 2-1` 부분은 반복 종료 조건이다. 크게 두가지인데, 첫번째 `if` 구문은 모든 음식재료가 `K` 이상이 되는 경우이고, 두번째 `if` 구문은 더 이상 음식섞기가 불가능할 때이다.

첫번째 종료조건을 보면, 단순히 `s` 리스트의 0 번째 요소가 `K` 이상인지만을 검사한다. heap 자료 구조상, 0 번째 요소가 제일 작은 값을 가지기 때문에, 0 번 인덱스 값이 `K` 이상이면 다른 요소 모두 `K` 이상이다.

두번째 종료조건을 보면, `s` 의 길이가 1 이 되는지를 검사하는데, 두가지 음식물을 섞어야하기 때문에 더이상 섞는 작업이 불가능함을 나타낸다. 이미 첫번째 종료조건을 만족시키지 못한상태에서 두번째 요소를 만족시킨다는 건, 하나밖에 없는 `s` 의 0 번째 요소 값이 `K` 미만이라는 뜻이고, 이는 곧 `K` 이상이 되도록 음식물 조합이 불가능하다는 뜻이므로, 문제에서 요구하는대로 -1 을 리턴한다.

## 참고

참고로 이 문제의 python 효율성 테스트는 너무 빡빡하게 설정한 것 같다. heapq 모듈을 사용하지 않고는 거의 불가능한 듯 보인다.
