## 문제

[프로그래머스, 키패드 누르기](https://programmers.co.kr/learn/courses/30/lessons/67256?language=python3)

주어진 번호대로 키를 누를 때, 어느 손으로 눌렀는지를 계속 추적해야 하는 문제다.

어떤 n 이라는 키를 눌러야 할 때, 어느손을 사용해야 하는지 아래와 같은 순서대로 판단한다.

- 왼손으로 눌러야만 하는 키인가? -> 왼손 사용
- 오른손으로 눌러야만 하는 키인가? -> 오른손 사용
- 상대거리 계산
  - 왼손이 가까운가? -> 왼손 사용
  - 오른손이 가까운가? -> 오른손 사용
    - 주 사용손이 왼손인가? -> 왼손 사용
    - -> 오른손 사용

또한 상대거리 계산 편의를 위해, 각각의 키의 좌표를 임의로 미리 설정하였다.

## 풀이

```py
def solution(numbers, hand):
    # 키패드와 상대좌표 keypad 에 할당
    # 누른손 누적값 a, 왼손 현재위치, 오른손 현재위치 초기화
    keypad = {
        '1': [0, 0], '2': [0, 1], '3': [0, 2],
        '4': [1, 0], '5': [1, 1], '6': [1, 2],
        '7': [2, 0], '8': [2, 1], '9': [2, 2],
        '*': [3, 0], '0': [3, 1], '#': [3, 2],
    }
    a, lh, rh = '', '*', '#'

    # numbers 순회, 누른손 판단하여 a 에 삽입, 순회 종료 후 a 리턴
    for n in numbers:
        n = str(n)
        # 왼/오른손으로만 누르는 키패드
        if n in ['1', '4', '7']: a, lh = a+'L', n
        elif n in ['3', '6', '9']: a, rh = a+'R', n
        
        # 나머지 키패드, 상대거리 계산하여 결정
        else:
            [y, x], [ly, lx], [ry, rx] = keypad[n], keypad[lh], keypad[rh]
            d = (abs(y-ly)+abs(x-lx)) - (abs(y-ry)+abs(x-rx))
            if d<0: a, lh = a+'L', n
            elif d>0: a, rh = a+'R', n
            else:
                if hand == 'left': a, lh = a+'L', n
                else: a, rh = a+'R', n
    return a
```
{:.python}

키마다 임의의 좌표를 `keypad` 딕셔너리로 설정해두었다. 그리고 상대거래를 계산하려면, 현재의 왼/오른손이 어디 위치에 있는지도 알아야하므로, 현재 손들의 위치를 가리키는 `lh`, `rh` 변수도 생성하고 초기값을 주었다.

`numbers` 를 `n` 으로 순회하면서, 위에서 언급한 순서대로 판단을 하기 위해, 중첩 if 구문을 사용하였다.
