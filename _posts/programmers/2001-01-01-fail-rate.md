## 문제

[프로그래머스, 실패율](https://programmers.co.kr/learn/courses/30/lessons/42889?language=python3)

문제를 보면 실패율을 계산, 그리고 이를 정렬하여 최종리턴, 이 두가지를 해결해야 하는 문제다.

## 풀이

```py
def solution(N, stages):
    # 1. {스테이지: 실패율, ... } 형태의 자료를 담을 a 딕셔너리 생성
    a = {}
    
    # 2. 스테이지를 나타내는 i 반복, 실패율 계산하여 a 에 삽입
    for i in range(1, N+1):
        x = stages.count(i)
        y = [e for e in stages if e >= i].__len__()
        a[i] = x/y if y else 0
        
    # 3. "실패율 내림차순, 스테이지 오름차순" 순서로 정렬한 리스트 리턴
    return sorted(a, key=lambda e: (-a[e], e))
```

python 은 다른 언어에 비해 정렬을 매우 쉽게 구현할 수 있는 것 같다. `key=함수` 형태로 비교함수를 넘기는데, 튜플 자료형 리턴만으로 비교 우선순위를 간략하게 표현할 수 있도록 되어있다.

## 수정

다른풀이를 보다보니, 위 코드보다 더 효율적으로 작동할 것 같은 코드가 눈에 띄였다. 반복 횟수를 줄인 코드다.

원코드를 보면, `# 2` 부분에서 `x` 를 구하기 위해 `stages` 를 한번 순회하는데, `y` 를 구하기 위해서 또 순회를 한다. 만일 `stages` 의 길이가 매우 길다면 실행속도가 그만큼 떨어지게 될 것이다.

아래는 원코드의 이중반복 문제를 수정한 코드다.

```py
# ... 생략

# 2
y = len(stages)
for i in range(1, N+1):
    x = stages.count(i)
    # y = [e for e in stages if e >= i].__len__()
    a[i] = x/y if y else 0
    y -= x

# ... 생략
```

실패율의 분모에 해당하는 `y` 를 `for` 반복문 바깥에서 `stages` 길이로 할당을 해둔다. 반복턴 마지막마다 `x` 만큼 씩 차감하는데, 이러면 스테이지 `i` 보다 크거나 같은 요소의 개수와 일치하게 된다.

물론 직관적이지는 않지만 전체 코드의 반복 횟수를 줄일 수 있어 속도가 더 빨라진다. 실제로 원코드와 수정코드 실행 속도를 비교해보면 더 명확하다.
