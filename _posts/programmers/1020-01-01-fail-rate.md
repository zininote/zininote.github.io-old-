## 문제

[프로그래머스, 실패율](https://programmers.co.kr/learn/courses/30/lessons/42889)

문제에서 요구하는 그대로 실패율을 계산하고, 정렬을 하면 된다.

스테이지를 순회하면서, 스테이지별로 실패율을 계산, 별도의 배열에 담은 다음, 해당 배열을 문제에서 요구한대로 실패율 내림차순, 스테이지 오름차순 순으로 정렬한뒤 리턴하는 식으로 구하였다.

## 풀이

```js
function solution(N, stages) {
    // 실패율 계산, [{'s': 스테이지, 'r': 실패율}, ... ] 형태로 s 에 할당
    var s = [];
    for(var x = 1; x <= N; x++) {
        var n = stages.filter(e => e == x).length;
        var d = stages.filter(e => e >= x).length;
        s.push({'s': x, 'r': (d ? n/d : 0)});
    }
    
    // 실패율 내림차순, 스테이지 오름차순 순으로 s 정렬
    s.sort(function(x, y) {
        return x['r'] != y['r'] ? -(x['r']-y['r']) : x['s']-y['s'];
    });
    
    // s 에서 스테이지만 추출하여 리턴
    return s.map(e => e['s']);
}
```
{:.javascript}

```py
def solution(N, stages):
    # 실패율 계산, [{'s': 스테이지, 'r': 실패율}, ... ] 형태로 s 에 할당
    s = []
    for x in range(1, N+1):
        n = stages.count(x)
        d = [e for e in stages if e >= x].__len__()
        s.append({'s': x, 'r': n/d if d else 0})
    
    # 실패율 내림차순, 스테이지 오름차순 순으로 s 정렬
    s.sort(key=lambda e: (-e['r'], e['s']))
    
    # s 에서 스테이지만 추출하여 리턴
    return [e['s'] for e in s]
```
{:.python}

Javascript 코드에서 실패율 계산할 때는 `filter` 함수를, 마지막에 `s` 배열에서 스테이지만 추출할 땐 `map` 함수를 사용하였다. Python 도 `filter` 와 `map` 함수를 빌트인으로 제공하지만, 개인적으로는 Python 의 리스트 comprehension 표현식이 너무 편리해서 이를 사용하였다.

또 하나의 큰 차이는 정렬이다. Python 은 `key` 인수에 튜플을 결과로 반환하는 함수를 적용하면, 튜플의 순서대로 정렬을 해준다. 개인적으로는 아주 직관적이고 편리하다고 본다.

Javascript 정렬은 인수로 줄 수 있는 비교함수가 다소 이해하기 불편하다. 위 코드처럼 x, y 를 순서대로 인수로 적용할 때, 음수를 리턴하면 x -> y 순으로, 양수를 리턴하면 y -> x 순으로 정렬한다. 비교함수에 대해서는 [MDN 공식문서](https://developer.mozilla.org/ko/docs/Web/JavaScript/Reference/Global_Objects/Array/sort)를 참고해보기 바란다.

## 참고

참고로, 다른풀이를 보면, 위 코드 보다 더 효율적인 코드가 있다. 실패율 계산을 더 빠르게 할 수 있는 방법인데, 일단 위 두 코드의 실패율 계산 의사코드부터 살펴보자.

```pseudo
스테이지를 1 부터 N 까지 순회
    n <- stages 배열 순회, 요소값 == 스테이지인 요소개수 합산
    d <- stages 배열 순회, 요소값 >= 스테이지인 요소개수 합산
    s <- 실패율 n/d 계산, {'s': 스테이지, 'r': 실패율} 생성
```

아래는 다른풀이의 의사코드로, d 를 순회할 필요가 없어 더 빠른 계산이 가능하다.

```pseudo
d <- stages 전체 요소개수
스테이지를 1 부터 N 까지 순회
    n <- stages 배열 순회, 요소값 == 스테이지인 요소개수 합산
    s <- 실패율 n/d 계산, {'s': 스테이지, 'r': 실패율} 생성
    d <- d - n
```

`d` 는 실패율 계산의 분모로, 해당 스테이지에 도달한 플레이어의 수다. 기존 코드에서는 직접 계산을 하지만, 아래 다른풀이코드에서는 먼저 `d` 에 전체 요소개수를 넣은다음, 해당 스테이지에 해당하는 수만큼을 차감하는 식으로 계산한다.

스테이지 반복이 1 부터 N 까지이므로, 전체에서 1 스테이지 플레이어 수를 빼면, 2 스테이지 이상 도달을 해본 플레이어 수가 된다. 다소 직관적이지는 않지만, 결국 스테이지 도달한 플레이어 수를 계산하고 있으며, 순회라는 방식을 사용하지 않으므로 속도가 빠를 수밖에 없다.
